Enrichening Inform 7 Interactive Fiction with Vorple

About

Introduction to Vorple

Vorple is a user interface library for interactive literature. Its goal is to let online hypertext fiction and parser interactive fiction systems break out of their sandboxes and let story authors use the full scale of the vast possibilities of technologies that are commonly used in the World Wide Web.

Most interactive fiction development systems produce story files that are played with a specialized interpreter. The interpreter model guarantees high portability and security, but comes with the cost of binding the stories inside the boundaries of the virtual machine. It has been very difficult if not impossible for an author to accomplish something that the development system has not provided.

Vorple changes the game by letting the story file communicate with the web interpreter. An author might use Vorple just to theme their story with one of the provided custom themes, or make a multimedia spectacle with pictures, video, sound effects and interactive elements, or anything in between or beyond.

Vorple comes with a custom Parchment web interpreter that can communicate with the story file, a number of Inform 7 extensions that provide the new features, and some custom themes.

The project's headquarters are at www.vorple-if.com where you'll find all the official material and downloads. JavaScript developers who want to dig into the innards of the system can find the source code at Github.

The library and all official material are open source and free for anyone to use for any purpose. (Mandatory footnote: Do still read all the licenses if you want to use it commercially or as a part of a closed system. Some third party themes, for example, are subject to slightly more restrictive licenses.)

Known limitations

Vorple is experimental software and in early stages of development. There are a number of known issues that will be fixed in later releases:

  • Single-character input is not supported, most notably pausing the story ("press any key to continue") does not work.
  • Status line isn't available, including features that are built on the status line like in-game menus.
  • Command history (browsing old commands with the up arrow key) isn't available.
  • Undoing is not reliable if the browser communicates with the story file using hidden commands.
  • Doesn't work with Internet Explorer 8. Not tested with earlier version of Internet Explorer.
  • Sound does not work in all browsers.
At the moment only Z-machine is supported. Glulx support is waiting for the next version of the Glulx specification. This is to avoid creating a competing non-standard specification.

There's a larger library running below the Inform layer and there isn't yet extension coverage for all of it. If you're feeling adventurous you can try to invoke the methods manually (with no guarantees of it working, of course). They are described in the API and in the Undum sister manual.

Installation and usage

Getting started

First of all, you need the Inform 7 authoring system installed and the skills to use it. Inform comes with comprehensive documentation, and other learning resources are listed here.

Once you're ready with Inform, continue with Vorple:

  1. Download the zip file that includes the interpreter template and the Inform 7 extensions. The extensions can also be downloaded individually, but they won't do anything without the interpreter.
  2. Install the interpreter template by copying the Vorple directory to the Templates directory either in the Inform's library directory or in an individual project's Materials directory, as per chapter 23.11 in Writing with Inform.
  3. Install the extensions (File→Install extension... in the Inform IDE menu). You must have at least the Vorple Core extension installed.
  4. Read the extension documentation (Installed Extensions link in the IDE documentation page). Read at least the two first chapters of the Vorple Core extension's documentation.
  5. Add a Release along with the "Vorple" interpreter line to the story source.
  6. Click the "Release" button, then "browse play-in-browser page" link in the release report page.

Remember that the interpreter running inside the Inform IDE is not a Vorple-capable interpreter, so you won't see any of the new features when you run the story in it. You need to release the story and play in a web browser instead.

Custom CSS and JavaScript

The interpreter tries to load a vorple.custom.css CSS file and a vorple.custom.js JavaScript file if they exist. Those are the best places to put custom CSS and JavaScript code.

Place the file(s) in the Materials folder of the Inform project and include them in the release: 
Release along with a file of "Custom CSS" called "vorple.custom.css".
Release along with a file of "Custom JavaScript" called "vorple.custom.js".

Support

The best place to look for assistance is the community forum at intfiction.org. Vorple's author frequents the forum, as do other experienced authors who are happy to answer civilized help requests about authoring with Inform, Vorple or other systems.

Bugs can be reported at the project's Github pages or by sending mail to bugs@vorple-if.com.
